## Integration Findings

### Critical Breaks

**INT-01: Payload Key Mismatch (Phase 2 → Phase 3)**
- Node.js `download.ts` enqueues `{filingId, filingType, dealId, firmIds}` (camelCase)
- Python `worker.py` reads `{filing_id, filing_type, deal_id, firm_ids}` (snake_case)
- Every `llm_extract` job silently receives empty values and extracts nothing
- **Fix:** Change Python `.get()` calls to use camelCase keys, or change Node.js to enqueue snake_case

**INT-02: Missing /api/deals/:id/clauses Route**
- Frontend `getClauses()` calls `/api/deals/${dealId}/clauses`
- No such route exists in `apps/api/src/routes/deals.ts`
- Silently falls back to `MOCK_CLAUSES` — extracted data in DB is unreachable
- **Fix:** Add Hono route querying `schema.clauses` filtered by dealId

### Functional Gaps

**INT-03: Missing /api/deals/:id/market-snapshots Route**
- `getMarketSnapshots()` calls non-existent endpoint; spread chart uses mock data
- Scheduled for Phase 5 (Market Data) — not a current-phase blocker

**INT-04: Redis TLS Mismatch**
- Python worker defaults to `redis://localhost:6379` (non-TLS)
- Upstash requires `rediss://` scheme
- **Fix:** Update `.env.example` to document `rediss://` format

### Architecture Gaps

**INT-05: Alert Rules in localStorage**
- DB `alert_rules` table exists but has no API route
- Frontend writes to `localStorage` instead — settings are per-browser, not per-team
- Scheduled for Phase 5 (Alert Delivery) — documented for future
