**1. S-4/DEFM14A Pipeline (`pipelines/s4_defm14a.py`)**:

This is the richest pipeline — S-4 and DEFM14A filings contain the full merger agreement terms.

```python
import asyncio
import json
from pathlib import Path
import langextract as lx
from langextract.data import ExampleData, Extraction
from db import insert_clause, mark_filing_extracted, insert_filing_summary
from summaries.analyst_summary import generate_headline_summary, generate_section_summary

PROMPT = """
Extract M&A deal clauses from this merger agreement proxy filing (S-4 or DEFM14A).
For each clause, extract the EXACT VERBATIM TEXT as it appears in the document — do not paraphrase.

Classify each clause into one of these types:
- TERMINATION_FEE: Fee payable by the target company upon deal termination
- REVERSE_TERMINATION_FEE: Fee payable by the acquirer upon deal termination
- MAE: Material Adverse Effect definition and exclusions
- REGULATORY_EFFORTS: Regulatory approval covenants (HSR, FTC, DOJ, international)
- LITIGATION_CONDITION: Conditions related to pending litigation
- FINANCING_CONDITION: Conditions related to financing availability
- GO_SHOP: Period during which target can solicit competing offers
- TICKING_FEE: Fee that increases over time if deal is delayed
- HELL_OR_HIGH_WATER: Obligation to accept regulatory remedies
- SPECIFIC_PERFORMANCE: Right to seek specific performance vs. damages
- NO_SHOP: Restriction on soliciting competing offers
- MATCHING_RIGHTS: Right to match competing offers
- OTHER: Any other material clause not in the above categories

For each extraction, include these attributes:
- "amount": Dollar amount or percentage if applicable (e.g., "$2,100,000,000" or "4.2%")
- "direction": Who pays/benefits (e.g., "company_to_parent", "parent_to_company")
- "conditions": Key conditions or triggers (brief description)
- "section_ref": Section reference in the filing (e.g., "Section 8.1(c)")
""".strip()

# Model: Gemini 2.5 Pro for S-4/DEFM14A (complex legal language) — PINNED
MODEL_ID = "gemini-2.5-pro"

# Chunking config for legal documents (DataCamp recommendation)
EXTRACTION_PASSES = 5
MAX_WORKERS = 10
MAX_CHAR_BUFFER = 800
```

- Load few-shot examples from `examples/s4_examples.jsonl` using `load_examples()` function
- Define `async def run_s4_pipeline(filing_id, deal_id, firm_ids, raw_content)`:
  - Call `lx.extract()` wrapped in `asyncio.to_thread()` (CRITICAL — Pitfall 2: lx.extract is blocking)
  - For each extraction in result.extractions:
    - Map `extraction.extraction_class` to ClauseType
    - Store `extraction.extraction_text` as `verbatimText`
    - Store `extraction.char_interval.start_pos:end_pos` as `sourceLocation` (format: "52340:52412")
    - Calculate confidence score from alignment status: ALIGNED=0.9, APPROXIMATE=0.6, FAILED=0.3
    - Call `insert_clause()` to write to DB
  - After all clauses written, call `mark_filing_extracted(filing_id)`
  - Generate summaries: call `generate_headline_summary()` and `generate_section_summary()` (section summaries only for S-4/DEFM14A per locked decision)
  - Call `insert_filing_summary()` to store summaries

**Confidence score calculation** (Claude's discretion):
```python
def get_confidence_score(extraction) -> float:
    if not extraction.char_interval:
        return 0.3  # No grounding = low confidence
    if extraction.char_interval.start_pos is not None and extraction.char_interval.end_pos is not None:
        return 0.9  # Full grounding = high confidence
    return 0.6  # Partial grounding = medium confidence
```

**CRITICAL — Verbatim text in examples (Pitfall 5):**
Few-shot examples MUST use exact copy-paste verbatim text from real filings. LangExtract uses string alignment to map `extraction_text` back to source positions. Paraphrased text breaks alignment and produces null `char_interval`.

import os

**Two functions:**

**`async def generate_headline_summary(clauses, filing_type, filing_metadata) -> str`:**
- Takes extracted clauses (list of dicts with type, verbatim_text, attributes) + filing metadata
- Prompt: "Generate a 2-3 sentence analyst-note style summary of this {filing_type} filing. Highlight what matters most for a merger-arb analyst: key deal terms, risk flags, unusual provisions. Be factual but opinionated — flag things that deviate from standard M&A practice. Example tone: 'S-4 reveals $2.1B reverse termination fee (4.2% of deal value — above median). MAE clause notably excludes pandemic. Watch: shareholder vote March 15.'"
- Returns 2-3 sentence headline summary
- Wrap Gemini call in `asyncio.to_thread()` (or use async client if available)

**`async def generate_section_summary(clauses_by_category, filing_type) -> str`:**
- Takes clauses grouped by category (Termination Provisions, Conditions, Protective Provisions)
- Only called for S-4/DEFM14A (per locked decision)
- Generates expandable section-level summaries for each category
- Returns multi-paragraph summary with section headers

**Delta-aware summaries** (per locked decision):

**`async def generate_delta_summary(new_clauses, previous_clauses, filing_type) -> str`:**
- Compare new extraction against previous extraction for the same deal
- Highlight what changed: "Revised S-4 increases termination fee from $1.8B to $2.1B"
- If no prior extraction exists, fall back to standard headline summary
- Query previous clauses from DB (via `db.py` helper)

**Risk flag detection** (inline in summary prompt):
- Flag termination fees above 3% of deal value as "above median"
- Flag MAE clauses that exclude specific categories (pandemic, war, etc.)
- Flag go-shop periods shorter than 30 days
- Flag hell-or-high-water obligations
- These are prompt-level instructions, not code logic

**Error handling:**
- If Gemini API call fails, log error and return a fallback summary: "Summary generation failed — review extracted clauses directly."
- Do NOT block filing extraction on summary failure — summaries are enhancement, not critical path
