**1. Update `apps/api/src/edgar/download.ts`:**

After the `adminDb.update(schema.filings).set(...)` call (line 60-67), add an `llm_extract` job enqueue:

```typescript
import { ingestionQueue } from '../queues/ingestion.js';

// ... existing download code ...

// After storing rawContent, enqueue extraction job
// NOTE: Do NOT inline rawContent — Python worker fetches from DB (Pitfall 1)
await ingestionQueue.add('llm_extract', {
  filingId,
  filingType: /* need to get filingType from DB or job data */,
  dealId: /* need to get dealId from job data or DB */,
  firmIds: /* need to get firmIds — may require deal-matcher lookup */,
});
```

**To get filingType, dealId, firmIds:** The download handler currently only receives `filingId`, `accessionNumber`, `cik`, `primaryDocument`. It does NOT have `filingType`, `dealId`, or `firmIds`.

**Solution:** Query the filing row after update to get `filingType` and `dealId`:
```typescript
const [filingRow] = await adminDb
  .select({ filingType: schema.filings.filingType, dealId: schema.filings.dealId })
  .from(schema.filings)
  .where(eq(schema.filings.id, filingId));
```

For `firmIds`, query deals table:
```typescript
const firmIds: string[] = [];
if (filingRow.dealId) {
  const dealRows = await adminDb
    .select({ firmId: schema.deals.firmId })
    .from(schema.deals)
    .where(and(eq(schema.deals.id, filingRow.dealId), isNull(schema.deals.deletedAt)));
  firmIds.push(...dealRows.filter(d => d.firmId).map(d => d.firmId!));
}
```

Then enqueue:
```typescript
await ingestionQueue.add('llm_extract', {
  filingId,
  filingType: filingRow.filingType,
  dealId: filingRow.dealId,
  firmIds,
});
console.log(`[edgar_download] Enqueued llm_extract for filing ${accessionNumber}`);
```
