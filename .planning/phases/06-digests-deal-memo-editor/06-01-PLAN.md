Output: Digest cron jobs, react-email templates, digest preference CRUD, Settings > Digests tab.

    **2. Schema migration — add `digest_preferences` table to schema.ts:**
    ```typescript
    export const digestPreferences = pgTable(
      'digest_preferences',
      {
        id: uuid('id').defaultRandom().primaryKey(),
        firmId: uuid('firm_id').references(() => firms.id).notNull(),
        userId: uuid('user_id').notNull(),
        dailyEnabled: boolean('daily_enabled').notNull().default(true),
        weeklyEnabled: boolean('weekly_enabled').notNull().default(true),
        suppressWeekend: boolean('suppress_weekend').notNull().default(false),
        ...timestamps,
      },
      () => firmIsolationPolicies(),
    );
    ```
    Run `pnpm drizzle-kit push` from apps/api to apply.

    **3. Create `apps/api/src/digests/daily-digest.ts`:**
    - Export `queryOvernightEvents(firmId: string)` — queries events table WHERE firmId, createdAt > yesterday 8 AM ET (use Intl.DateTimeFormat for ET, same pattern as market-poller.ts), severity IN ('CRITICAL', 'WARNING'), ordered by materialityScore DESC.
    - Return array of `{ id, title, type, subType, severity, materialityScore, dealName, sourceUrl, createdAt }` with deal name joined from deals table.

    **4. Create `apps/api/src/digests/weekly-digest.ts`:**
    - Export `queryWeeklyDealChanges(firmId: string)` — queries events grouped by dealId for the past 7 days. For each deal: event count by severity, latest spread from market_snapshots, deal status, any status changes.
    - Return array of `{ dealId, dealName, status, eventCount, criticalCount, warningCount, latestSpread, statusChanged }`.

    **5. Create react-email templates:**

    `apps/api/src/digests/templates/daily-digest.tsx`:
    - Dark theme (bg #18181b, surface #27272a, text #fafafa, muted #a1a1aa, border #3f3f46).
    - Aurora palette accents: amber (#f5a623) for header, severity colors for event badges.
    - Layout: j16z logo text header -> date range -> event list grouped by deal -> footer with unsubscribe link.
    - Each event row: severity badge (colored dot), title, deal name, time.
    - Export `DailyDigestEmail` component with props: `{ events, dateRange, userName }`.

    `apps/api/src/digests/templates/weekly-digest.tsx`:
    - Same dark theme and Aurora palette.
    - Layout: j16z header -> "Week of {date}" -> deal summary table (columns: Deal, Status, Events, Spread, Alert) -> per-deal rows with color-coded status badges -> footer.
    - Export `WeeklyDigestEmail` component with props: `{ deals, weekRange, userName }`.

    **6. Create `apps/api/src/digests/digest-handler.ts`:**
    - Export `handleDigestDaily(job: Job)` and `handleDigestWeekly(job: Job)`.
    - Daily handler flow:
      1. Get all firms.
      2. For each firm, get firm_members.
      3. For each member, check digest_preferences (default: enabled if no row exists).
      4. Check suppressWeekend: if today is Sat/Sun (ET) and user has suppressWeekend=true, skip.
      5. Call `queryOvernightEvents(firmId)`. If empty, skip (no empty digests).
      6. Render DailyDigestEmail via `render()` from @react-email/components.
      7. Send via getResend().emails.send() with from: 'j16z Digests <digests@j16z.com>', subject: `j16z Daily Digest -- {date}`.
    - Weekly handler: same loop but uses queryWeeklyDealChanges, WeeklyDigestEmail template, fires Friday only.
    - Import getResend pattern from email-delivery.ts (or extract to shared util).

    **7. Register digest cron schedules in scheduler.ts:**
    - Add to SCHEDULE_CONFIG: `digest_daily: process.env.CRON_DIGEST_DAILY ?? '0 12 * * *'` (12:00 UTC = 8:00 AM ET during EDT, 7:00 AM during EST; but use tz option for accuracy).
    - Add to SCHEDULE_CONFIG: `digest_weekly: process.env.CRON_DIGEST_WEEKLY ?? '0 21 * * 5'` (21:00 UTC Friday = 5:00 PM ET).
    - Register with `tz: 'America/New_York'` in the scheduler config object: `{ pattern: '0 8 * * *', tz: 'America/New_York' }` for daily, `{ pattern: '0 17 * * 5', tz: 'America/New_York' }` for weekly.

    **8. Register handlers in worker.ts:**
    - Import `handleDigestDaily, handleDigestWeekly` from digests/digest-handler.
    - Add to handlers registry: `digest_daily: handleDigestDaily, digest_weekly: handleDigestWeekly`.

    **9. Create digest preferences API route `apps/api/src/routes/digest-preferences.ts`:**
    - GET /api/digest-preferences — returns current user's preferences (or defaults if no row).
    - PUT /api/digest-preferences — upserts preferences for current user (dailyEnabled, weeklyEnabled, suppressWeekend).
    - Uses adminDb with explicit firmId WHERE (defense-in-depth pattern).
    - Register route in routes/index.ts.
