    - `"use client"` directive.
    - Export `SectionRefreshButton` component. Props: `{ editor: Editor; sectionTitle: string; dealId: string }`.
    - Refreshable sections per CONTEXT.md locked decision: "Deal Terms", "Regulatory Status", "Litigation", "Spread History".
    - On click:
      1. Save cursor position (`editor.state.selection`).
      2. Fetch fresh data based on sectionTitle:
         - "Deal Terms": `getDeal(dealId)` -> rebuild table nodes via scaffold helpers.

      5. Build replacement content (heading + new section body) as ProseMirror nodes using `editor.schema.nodes`.
      6. Apply transaction: `editor.state.tr.replaceWith(startPos, endPos, replacementNodes)`.
      7. Dispatch transaction via `editor.view.dispatch(tr)`.
      8. Restore cursor: if cursor was inside refreshed section, place at end of section; otherwise restore original position.
    - Visual: small refresh icon (lucide-react RefreshCw) rendered inline next to H2 headings for refreshable sections.
    - Loading state: spinner replaces refresh icon during fetch.

      - Modified text: amber highlight.
