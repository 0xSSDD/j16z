1. **Create `apps/api/src/db/schema.ts`:**

Define reusable column helpers first:
```typescript
const timestamps = {
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
```

Define an RLS policy helper function that creates firm isolation policies for each table (SELECT, INSERT, UPDATE, DELETE). Use `sql\`(select ((auth.jwt() -> 'app_metadata')::jsonb ->> 'firm_id')::uuid)\`` pattern (wrapped in subquery for single-evaluation-per-statement optimization per research pitfall).

**Tables to define (all with UUIDs as primary keys):**

a. **`firms`** — id, name, timestamps. No firm_id (self-referencing would be circular). No RLS needed (admin-only access).

b. **`firm_members`** — id, firm_id (FK firms), user_id (uuid, references auth.users), role (text: 'admin' | 'member'), invited_by (uuid, nullable), timestamps. RLS: firm isolation on firm_id.

c. **`invites`** — id, firm_id (FK firms), email (text), role (text), token (text unique), expires_at (timestamp), accepted_at (timestamp nullable), created_by (uuid), timestamps. RLS: firm isolation.

d. **`deals`** — id, firm_id (FK firms), symbol (text), acquirer (text), target (text), deal_value (numeric), price_per_share (numeric), premium (numeric), current_price (numeric), gross_spread (numeric), annualized_return (numeric), status (text, default 'ANNOUNCED'), consideration_type (text), announced_date (date), expected_close_date (date), outside_date (date), p_close_base (numeric), p_break_regulatory (numeric), p_break_litigation (numeric), regulatory_flags (text[] array), litigation_count (integer default 0), spread_entry_threshold (numeric), size_bucket (text), is_starter (boolean default false), timestamps. RLS: firm isolation.

e. **`events`** — id, firm_id (FK firms), deal_id (FK deals), type (text), sub_type (text), title (text), description (text), source (text), source_url (text), timestamp (timestamptz), materiality_score (integer), severity (text), metadata (jsonb), timestamps. RLS: firm isolation.

f. **`filings`** — id, firm_id (FK firms), deal_id (FK deals, nullable), accession_number (text unique), filing_type (text), filer_name (text), filer_cik (text), filed_date (date), raw_url (text), raw_content (text, nullable — stores raw filing text), extracted (boolean default false), timestamps. RLS: firm isolation.

g. **`clauses`** — id, firm_id (FK firms), deal_id (FK deals), filing_id (FK filings), type (text), title (text), summary (text), verbatim_text (text), source_location (text), extracted_at (timestamptz), timestamps. RLS: firm isolation.

h. **`market_snapshots`** — id, firm_id (FK firms), deal_id (FK deals), timestamp (timestamptz), current_price (numeric), target_price (numeric), acquirer_price (numeric), gross_spread (numeric), annualized_return (numeric), volume (integer), timestamps. RLS: firm isolation.

i. **`news_items`** — id, firm_id (FK firms), deal_id (FK deals, nullable), title (text), source (text), url (text), published_at (timestamptz), summary (text), timestamps. RLS: firm isolation.

j. **`watchlists`** — id, firm_id (FK firms), name (text), description (text nullable), created_by (uuid), timestamps. RLS: firm isolation.

k. **`watchlist_deals`** — id, watchlist_id (FK watchlists), deal_id (FK deals), added_at (timestamptz default now), added_by (uuid). RLS: join through watchlist's firm isolation.

l. **`alert_rules`** — id, firm_id (FK firms), deal_id (FK deals, nullable — null means global), user_id (uuid), name (text), threshold (integer), channels (text[] — e.g. ['email', 'slack', 'webhook']), webhook_url (text nullable), is_active (boolean default true), timestamps. RLS: firm isolation.

m. **`audit_log`** — id, firm_id (FK firms), user_id (uuid), entity_type (text), entity_id (uuid), action (text — 'create', 'update', 'delete', 'restore'), changes (jsonb), timestamp (timestamptz default now). RLS: firm isolation. No deleted_at (audit logs are immutable).

**RLS policy pattern for each entity table:**
- Enable RLS on the table via `.enableRLS()` on the pgTable
- SELECT: `using: sql\`firm_id = (select ((auth.jwt() -> 'app_metadata')::jsonb ->> 'firm_id')::uuid)\``
- INSERT: `withCheck: sql\`firm_id = (select ((auth.jwt() -> 'app_metadata')::jsonb ->> 'firm_id')::uuid)\``
- UPDATE: same as SELECT using + INSERT withCheck
- DELETE: same as SELECT using
- All policies apply to role `authenticated`

**IMPORTANT per CONTEXT.md locked decisions:**
- ALL entities get soft deletes (`deleted_at` column) — audit_log is the only exception (immutable)
- `is_starter` boolean on deals for seed data badge distinction
- Roles are `admin` and `member` on firm_members

2. **Create `apps/api/src/db/index.ts`:**
   - Two Drizzle clients per Pattern 7 from RESEARCH.md:
     - `db`: User-facing, respects RLS. Uses `DATABASE_URL` (pooled connection, port 6543). `{ prepare: false }` required.
     - `adminDb`: Admin/service-role, bypasses RLS. Uses `SUPABASE_DB_URL_SERVICE_ROLE` (direct connection). `{ prepare: false }` for consistency.
   - Both import `* as schema` from `./schema`
   - Export both clients

**CRITICAL anti-patterns to avoid (from RESEARCH.md):**
- Do NOT use `auth.uid()` for firm isolation — use `app_metadata.firm_id` from custom access token hook
- Do NOT call `auth.jwt()` multiple times — wrap in subquery `(select auth.jwt()...)`
- Do NOT use `prepare: true` — Supabase pooler breaks prepared statements
