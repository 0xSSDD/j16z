  <action>
1. **Create `apps/api/src/middleware/auth.ts`:**
   - Import `createMiddleware` from `hono/factory`
   - Import `jwtVerify`, `createRemoteJWKSet` from `jose`
   - Construct JWKS URL: `${process.env.SUPABASE_URL}/auth/v1/.well-known/jwks.json`
   - Create JWKS instance using `createRemoteJWKSet(new URL(JWKS_URL))` — do this once at module level, not per-request
   - The middleware:
     a. Reads `Authorization` header, checks for `Bearer ` prefix
     b. If missing/malformed: return `c.json({ error: 'Missing or invalid authorization header' }, 401)`
     c. Extract token (slice after 'Bearer ')
     d. Call `jwtVerify(token, JWKS)` in try-catch
     e. On success: set `c.set('jwtPayload', payload)` and call `await next()`
     f. On failure: return `c.json({ error: 'Invalid or expired token' }, 401)`
   - Export `authMiddleware`

   **Key detail per RESEARCH.md:** Use `createRemoteJWKSet` (not a static secret) because Supabase uses RS256 asymmetric keys with key rotation.

2. **Create `apps/api/src/middleware/firm-context.ts`:**
   - Import `createMiddleware` from `hono/factory`
   - The middleware (runs AFTER auth):
     a. Read `c.get('jwtPayload')` from context (set by auth middleware)
     b. Extract `firm_id` from `payload.app_metadata?.firm_id`
     c. Extract `user_id` from `payload.sub`
     d. If `firm_id` is missing: return `c.json({ error: 'No firm associated with this account. Complete onboarding first.' }, 403)`
     e. Set `c.set('firmId', firm_id)` and `c.set('userId', user_id)`
     f. Call `await next()`
   - Export `firmContextMiddleware`

   **Defense-in-depth per RESEARCH.md open question 2:** Route handlers should ALSO add `where firm_id = ?` clauses using `c.get('firmId')` in addition to relying on RLS. Belt-and-suspenders for financial data.

3. **Update `apps/api/src/index.ts`:**
   - Import auth and firm-context middleware
   - Add middleware chain to the `/api` basePath BEFORE routes:
     ```typescript
     api.use('/*', authMiddleware);
     api.use('/*', firmContextMiddleware);
     ```
   - Routes are mounted AFTER middleware so every /api/* request is auth-protected

4. **Update route handlers in `deals.ts`, `events.ts`, `watchlists.ts`:**
   - Replace `adminDb` usage with `db` (RLS-respecting client)
   - Before each query, set the JWT context on the database connection for RLS. Since Drizzle with postgres.js requires explicit transaction-scoped `SET LOCAL` for RLS, use a helper:
     ```typescript
     // For RLS to work, we need to set the JWT in a transaction
     // Alternative: Use adminDb with explicit firm_id WHERE clause (defense-in-depth)
     ```
   - For this plan, use `adminDb` with explicit `where(eq(schema.deals.firmId, c.get('firmId')))` as the primary access pattern. This provides defense-in-depth with application-level scoping. RLS acts as the hard safety net.
   - Add `and(eq(schema.deals.firmId, firmId), isNull(schema.deals.deletedAt))` to every query WHERE clause
   - For INSERT operations, always set `firmId` from `c.get('firmId')` — never trust client-provided firm_id

   **Hono context typing:** Define a custom env type for type-safe context access:
   ```typescript
   type Env = {
     Variables: {
       jwtPayload: JWTPayload;
       firmId: string;
       userId: string;
     };
   };
   ```

     ```typescript
     async function authFetch(path: string, options: RequestInit = {}): Promise<Response> {
       const { createClient } = await import('@/lib/supabase/client');
       const supabase = createClient();
       const { data: { session } } = await supabase.auth.getSession();

       const headers = new Headers(options.headers);
       if (session?.access_token) {

       const response = await fetch(`${API_BASE_URL}${path}`, {
     ```
