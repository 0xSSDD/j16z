Add a `checkCanary` function to `rss-pollers.ts` and call it inside `handleAgencyRss` after parsing the feed, BEFORE processing items.

**checkCanary(config: PollConfig, currentCount: number): Promise&lt;void&gt;**:
- Read current ingestion_status row for `config.sourceName`: `adminDb.select().from(schema.ingestionStatus).where(eq(schema.ingestionStatus.sourceName, config.sourceName)).limit(1)`
- If `rows[0]` is undefined (first run, row does not exist yet): return early without firing canary. Let `handleAgencyRss` create the row via its existing `updateIngestionStatus` call at the end.
- Extract canary state from `metadata` jsonb, typed as: `{ canaryBaseline?: number; canaryZeroStreak?: number; lastBaselineUpdate?: string }`
- Defaults: `baseline = meta.canaryBaseline ?? 0`, `zeroStreak = meta.canaryZeroStreak ?? 0`
- If `baseline === 0`: this is the first canary check for this source. Store current count as baseline with `canaryZeroStreak: 0` in metadata via `adminDb.update(schema.ingestionStatus).set({ metadata: { canaryBaseline: currentCount, canaryZeroStreak: 0, lastBaselineUpdate: new Date().toISOString() } }).where(eq(schema.ingestionStatus.sourceName, config.sourceName))`. Return early â€” no alert on first baseline establishment.
- If `currentCount === 0` and `baseline > 5`:
  - Increment streak: `const newStreak = zeroStreak + 1`
  - If `newStreak >= 2` (2+ consecutive zero-item polls): fire canary via `updateIngestionStatus(config.sourceName, false, \`Canary alert: item count dropped from ${baseline} to 0 over ${newStreak} consecutive polls -- possible feed format change\`, 0)`
  - If `newStreak < 2`: just update metadata streak count, do NOT fire canary (avoids weekend false positives per RESEARCH.md pitfall 5). Update via `adminDb.update(schema.ingestionStatus).set({ metadata: { ...meta, canaryZeroStreak: newStreak } }).where(...)`
- If `currentCount > 0` and `currentCount < baseline * 0.2` (dropped >80%): fire canary immediately via `updateIngestionStatus(config.sourceName, false, \`Canary alert: item count dropped from ${baseline} to ${currentCount} -- possible feed format change\`, currentCount)`
- If `currentCount > 0` and count is healthy (>= baseline * 0.2): update baseline to current count, reset streak to 0 in metadata
- Log: `console.log(\`[${config.sourceName}] Canary check: ${currentCount} items (baseline: ${baseline})\`)`

**Integration point**: Call `await checkCanary(config, items.length)` inside `handleAgencyRss` right after the line `const items = (feed.items ?? []).slice(0, MAX_ITEMS_PER_POLL) as RssItem[];` and before `const deals = await loadActiveDeals();`.

```typescript
function getPacerCredentialHealth() {
  const lastChanged = process.env.PACER_PASSWORD_LAST_CHANGED; // ISO date e.g. '2026-01-01'
  if (!lastChanged) return { isExpiringSoon: false, daysUntilExpiry: null, expiryDate: null, lastChanged: null };
  const expiryDate = new Date(lastChanged);
  expiryDate.setDate(expiryDate.getDate() + 180); // PACER passwords expire every 180 days
  const daysUntilExpiry = Math.floor((expiryDate.getTime() - Date.now()) / (1000 * 60 * 60 * 24));
  return {
    isExpiringSoon: daysUntilExpiry <= 30,
    daysUntilExpiry,
    expiryDate: expiryDate.toISOString(),
    lastChanged,
  };
}
```
