  <name>Task 1: Canary monitor for agency RSS feeds</name>
  <files>apps/api/src/agency/rss-pollers.ts</files>
  <action>
Add a `checkCanary` function to `rss-pollers.ts` and call it inside `handleAgencyRss` after parsing the feed, BEFORE processing items.

**checkCanary(config: PollConfig, currentCount: number): Promise<void>**:
- Read current ingestion_status row for `config.sourceName`: `adminDb.select().from(schema.ingestionStatus).where(eq(schema.ingestionStatus.sourceName, config.sourceName)).limit(1)`
- Extract baseline from `metadata` jsonb: `{ canaryBaseline: number, canaryZeroStreak: number, lastBaselineUpdate: string }`
- If no baseline exists (first run or missing metadata): store current count as baseline with `canaryZeroStreak: 0` in metadata via `adminDb.update(schema.ingestionStatus).set({ metadata: { canaryBaseline: currentCount, canaryZeroStreak: 0, lastBaselineUpdate: new Date().toISOString() } }).where(eq(schema.ingestionStatus.sourceName, config.sourceName))`
- If `currentCount === 0` and baseline was `> 5`: increment `canaryZeroStreak` in metadata
  - If `canaryZeroStreak >= 2` (2+ consecutive zero polls): call `updateIngestionStatus(config.sourceName, false, 'Canary alert: item count dropped from ${baseline} to 0 over ${streak} consecutive polls -- possible feed format change', 0)`
  - If `canaryZeroStreak < 2`: just update metadata streak count, do NOT fire canary (avoids weekend false positives per RESEARCH.md pitfall 6)
- If `currentCount > 0` and current count is `< baseline * 0.2` (dropped >80%): fire canary: `updateIngestionStatus(config.sourceName, false, 'Canary alert: item count dropped from ${baseline} to ${currentCount} -- possible feed format change', currentCount)`
- If `currentCount > 0` and count is healthy (>= baseline * 0.2): update baseline to current count, reset `canaryZeroStreak` to 0
- Log canary checks: `console.log([${config.sourceName}] Canary check: ${currentCount} items (baseline: ${baseline})`

**Integration point**: Call `await checkCanary(config, items.length)` inside `handleAgencyRss` right after `const items = (feed.items ?? []).slice(0, MAX_ITEMS_PER_POLL)` and before the for-loop.

**2. PACER credential health function:**
```typescript
function getPacerCredentialHealth() {
  const lastChanged = process.env.PACER_PASSWORD_LAST_CHANGED; // ISO date string e.g. '2026-01-01'
  if (!lastChanged) return { isExpiringSoon: false, daysUntilExpiry: null, expiryDate: null, lastChanged: null };
  const expiryDate = new Date(lastChanged);
  expiryDate.setDate(expiryDate.getDate() + 180);
  const daysUntilExpiry = Math.floor((expiryDate.getTime() - Date.now()) / (1000 * 60 * 60 * 24));
  return {
    isExpiringSoon: daysUntilExpiry <= 30,
    daysUntilExpiry,
    expiryDate: expiryDate.toISOString(),
    lastChanged,
  };
}
```
