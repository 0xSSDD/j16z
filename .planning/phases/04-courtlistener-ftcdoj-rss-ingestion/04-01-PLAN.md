**event-factory.ts** — Create COURT events (mirrors `agency/event-factory.ts`):

- `createCourtEvent(params)` — inserts into `events` table with `type: 'COURT'`, `source: 'COURT_LISTENER'`
- Params interface: `{ firmId: string, dealId: string, subType: string, title: string, description: string, sourceUrl: string, materialityScore: number, severity: 'CRITICAL' | 'WARNING' | 'INFO', timestamp?: Date, metadata?: Record<string, unknown> }`
- `getCourtMaterialityScore(subType: string): number` — returns base score: INJUNCTION=90, COMPLAINT=85, SECOND_REQUEST=85, MOTION=70, CASE_DISCOVERED=50, DOCKET_ENTRY=50, default=50
- `getCourtSeverity(score: number): 'CRITICAL' | 'WARNING' | 'INFO'` — >=70 CRITICAL, >=50 WARNING, else INFO

**poller.ts** — `handleCourtListenerPoll(job: Job): Promise<void>`:
- Import `updateIngestionStatus` from `../agency/event-factory.js` (reuse, do not duplicate)
- Load active deals with non-null firmId: `adminDb.select().from(schema.deals).where(and(isNotNull(schema.deals.firmId), isNull(schema.deals.deletedAt)))`
- For each deal, search CourtListener v4: `GET ${COURTLISTENER_API_BASE}/api/rest/v4/search/?type=d&q=caseName:"${deal.target}" OR caseName:"${deal.acquirer}"&order_by=score+desc&count=5`
- Auth header: `Authorization: Token ${process.env.COURTLISTENER_API_TOKEN}`. User-Agent: `j16z-research/1.0 (compliance@j16z.io)`
- For each discovered docket, check if already subscribed: `adminDb.select().from(schema.events).where(and(eq(schema.events.type, 'COURT'), sql\`${schema.events.metadata} ->> 'courtlistenerDocketId' = ${String(docket.id)}\`)).limit(1)`
- If NOT subscribed, subscribe via v4 docket-alerts endpoint: `POST ${COURTLISTENER_API_BASE}/api/rest/v4/docket-alerts/` with `Content-Type: application/x-www-form-urlencoded` body `docket=${docket.id}`. CRITICAL: Use v4 endpoint, NOT v3 — v4 is confirmed current per 2026-03-13 research.

**courtlistener/webhook.ts** — BullMQ job handler (NOT the Hono route):
- `handleCourtListenerWebhook(job: Job): Promise<void>` — processes enqueued webhook payloads
- Parse `job.data` with `courtListenerWebhookSchema` from types.ts. Extract `idempotencyKey` from `job.data.idempotencyKey`.
- For each result in `payload.results`:
  - Look up existing COURT events to find associated deal: `SELECT events WHERE type='COURT' AND metadata->>'courtlistenerDocketId' = result.docket::text LIMIT 1`
  - If no matching event found, log warning and skip (orphaned webhook for untracked docket)
  - For each docket entry in `result.docket_entries`:
    - Check dedup: `SELECT events WHERE type='COURT' AND metadata->>'sourceGuid' = 'cl:entry:${entry.id}' LIMIT 1`
    - Infer subType from `entry.description` keywords: "injunction"->INJUNCTION, "motion"->MOTION, "complaint"->COMPLAINT, default->DOCKET_ENTRY
    - Create event via `createCourtEvent()` with the matched deal's firmId/dealId, appropriate materiality score from `getCourtMaterialityScore(subType)`, severity from `getCourtSeverity(score)`, metadata including `{ sourceGuid: 'cl:entry:${entry.id}', idempotencyKey, courtlistenerDocketId: result.docket }`
- Call `updateIngestionStatus('courtlistener', true, undefined, createdCount)` on success
- Wrap in try/catch with updateIngestionStatus on failure

**routes/webhooks.ts** — New Hono router (NO auth middleware):
- `export const webhooksRoutes = new Hono()`
- `.post('/courtlistener', handler)`:
  - Always log incoming IP: `const clientIp = c.req.header('cf-connecting-ip') ?? c.req.header('x-forwarded-for') ?? ''; console.log('[courtlistener webhook] incoming IP:', clientIp);`
  - IP allowlist: check `COURTLISTENER_IPS.has(clientIp)`, return `c.json({ error: 'Forbidden' }, 403)` if not in set
  - Idempotency-Key dedup: read `c.req.header('Idempotency-Key')`, query `events WHERE metadata->>'idempotencyKey' = key LIMIT 1`, return `c.json({ ok: true })` if already processed
  - Parse body: `courtListenerWebhookSchema.safeParse(await c.req.json())`, return 400 on failure
  - Enqueue BullMQ job: `const queue = new Queue('ingestion', { connection: redisConnection }); await queue.add('courtlistener_webhook', { ...parsed.data, idempotencyKey }, { attempts: 3 }); await queue.close();`
  - Return `c.json({ ok: true })` immediately — do NOT process synchronously (CourtListener has short timeout, retries up to 7 times)
- Import `COURTLISTENER_IPS`, `courtListenerWebhookSchema` from `../courtlistener/types.js`, `redisConnection` from `../queues/connection.js`, `Queue` from `bullmq`, `adminDb`/`schema` for dedup check, `sql` from `drizzle-orm`

**routes/index.ts** — Add webhooks export:
- Import `{ webhooksRoutes } from './webhooks.js'`
- Add `webhooks: webhooksRoutes` to the `apiRoutes` object

**index.ts** — Register webhook route BEFORE auth middleware (CRITICAL):
- Import is already handled via `apiRoutes.webhooks`
- Add `app.route('/api/webhooks', apiRoutes.webhooks);` AFTER the `app.get('/health', ...)` line and BEFORE `const api = app.basePath('/api');`
- This ensures CourtListener pushes are NOT blocked by authMiddleware. Without this, CourtListener gets 401, retries 7 times, then auto-disables the webhook.
