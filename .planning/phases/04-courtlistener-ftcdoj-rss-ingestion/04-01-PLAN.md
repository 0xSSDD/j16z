Create `apps/api/src/courtlistener/` directory with three files:

**types.ts** — Zod schemas for CourtListener API:
- `docketSearchResultSchema`: `{ id: number, case_name: string, docket_number: string | null, court: string, date_filed: string | null }` with `.passthrough()`
- `docketSearchResponseSchema`: `{ count: number, results: docketSearchResultSchema[] }` with `.passthrough()`
- `docketEntrySchema`: `{ id: number, description: string | null, entry_number: number | null, date_filed: string | null, recap_documents: [{ id: number, description: string | null }]? }` with `.passthrough()`
- `courtListenerWebhookSchema`: `{ webhook: { event_type: number, version: number }, payload: { results: [{ docket: number, case_name?: string, docket_entries?: docketEntrySchema[] }] } }` with `.passthrough()`
- `docketAlertSubscriptionSchema`: `{ id: number, docket: number, alert_type: number }` with `.passthrough()`
- Export `COURTLISTENER_API_BASE = 'https://www.courtlistener.com'`
- Export `COURTLISTENER_IPS = new Set(['34.210.230.218', '54.189.59.91'])` for webhook IP allowlist

**event-factory.ts** — Create COURT events (mirrors `agency/event-factory.ts`):
- `createCourtEvent(params)` — inserts into `events` table with `type: 'COURT'`, `source: 'COURT_LISTENER'`
- Params: `{ firmId, dealId, subType, title, description, sourceUrl, materialityScore, severity, timestamp?, metadata? }`
- `getCourtMaterialityScore(subType: string)` — returns base score per CLAUDE.md: injunctions=90, second_request=85, motions=70, docket_entry=50, default=50
- `getCourtSeverity(score: number)` — same getSeverity pattern: >=70 CRITICAL, >=50 WARNING, else INFO

**poller.ts** — `handleCourtListenerPoll(job: Job): Promise<void>`:
- Load all active deals via `adminDb.select().from(schema.deals).where(isNull(schema.deals.deletedAt))`
- For each deal, search CourtListener: `GET /api/rest/v4/search/?type=d&q=caseName:"${target}" OR caseName:"${acquirer}"&order_by=score+desc&count=5`
- Auth header: `Authorization: Token ${process.env.COURTLISTENER_API_TOKEN}`
- User-Agent: `j16z-research/1.0 (compliance@j16z.io)`
- For each discovered docket:
  - Check if already subscribed: `SELECT events WHERE type='COURT' AND metadata->>'courtlistenerDocketId' = docketId::text LIMIT 1`
  - If not subscribed, create docket-alert subscription: `POST /api/rest/v3/docket-alerts/` with body `docket=${docketId}` and `Content-Type: application/x-www-form-urlencoded` (v3 endpoint, NOT v4 per RESEARCH.md pitfall 7)
  - Create initial COURT event with `subType: 'CASE_DISCOVERED'`, `materialityScore: 50`, metadata containing `{ courtlistenerDocketId, courtlistenerAlertId, caseName, court, docketNumber }`
- Fetch recent docket entries for known dockets: `GET /api/rest/v4/docket-entries/?docket=${docketId}&order_by=-date_filed`
  - For each new entry (not already in events by sourceGuid `cl:entry:${entryId}`), create COURT event with appropriate subType inferred from description keywords (injunction, motion, complaint, etc.)
- Call `updateIngestionStatus('courtlistener', true, undefined, createdCount)` on success, `updateIngestionStatus('courtlistener', false, errorMessage)` on failure
- Wrap entire function in try/catch matching agency/rss-pollers.ts pattern

**webhook.ts** (in courtlistener/) — `handleCourtListenerWebhook(job: Job): Promise<void>`:
- Processes webhook payload from BullMQ job data
- Parse with `courtListenerWebhookSchema` from types.ts
- For each result in `payload.results`:
  - Look up existing COURT events with `metadata->>'courtlistenerDocketId' = result.docket::text` to find the associated deal
  - For each new docket entry in `result.docket_entries`:
    - Check dedup: `SELECT events WHERE type='COURT' AND metadata->>'sourceGuid' = 'cl:entry:${entry.id}' LIMIT 1`
    - Infer subType from entry description keywords: "injunction" -> INJUNCTION (90), "motion" -> MOTION (70), "complaint" -> COMPLAINT (85), default -> DOCKET_ENTRY (50)
    - Call `createCourtEvent()` with the matched deal's firmId and dealId
- Call `updateIngestionStatus('courtlistener', true, undefined, createdCount)`

**routes/webhooks.ts** — New Hono router (NO auth middleware):
- `POST /courtlistener`:
  - IP allowlist check: read `cf-connecting-ip` or `x-forwarded-for` header, verify against `COURTLISTENER_IPS` set. Return 403 if not allowed. Log the incoming IP always (for monitoring IP changes per RESEARCH.md pitfall 2).
  - Idempotency-Key dedup: read header, check `SELECT events WHERE metadata->>'idempotencyKey' = key LIMIT 1`, return `{ ok: true }` if already processed
  - Parse body with `courtListenerWebhookSchema.safeParse()`, return 400 on failure
  - Enqueue BullMQ job: `new Queue('ingestion', { connection: redisConnection }).add('courtlistener_webhook', parsed.data, { attempts: 3 })`
  - Return `{ ok: true }` immediately (do NOT process synchronously per RESEARCH.md anti-pattern)

**routes/index.ts** — Add `webhooksRoutes` export:
- Import `{ webhooksRoutes } from './webhooks.js'`
