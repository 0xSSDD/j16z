## Goal

Implement the full two-stage EDGAR ingestion pipeline: Stage 1 polls EDGAR APIs (CIK-based + EFTS broad scan), inserts filing metadata rows with `raw_content=null`, and enqueues download jobs. Stage 2 fetches filing HTML, converts to plain text, and stores it. Filings are matched to deals and firm-scoped Event records are created. After this plan, the worker processes `edgar_poll` and `edgar_download` jobs end-to-end.

## Context

### User Decisions (from CONTEXT.md — NON-NEGOTIABLE)

- **Hybrid discovery:** CIK-based watching for existing deals + periodic broad EFTS scan for new deals
- **Auto-create deals:** High-signal filings (S-4, DEFM14A) auto-create deal as ANNOUNCED; low-signal (8-K, 13D/13G) go to review queue (`status: 'pending_review'`)
- **Two-stage ingestion:** Poll captures metadata first; separate download job fetches content. Filing appears on deal board before content arrives.
- **HTML conversion:** Strip tags to plain text before storing; original always accessible via raw_url
- **Download failure handling:** BullMQ retries (3 attempts, exponential backoff); filing shows "content pending"; analyst can click EDGAR link
- **30-day backfill:** On first run, fetch 30 days of historical filings
- **Deduplication:** Accession number is UNIQUE in filings table; use `onConflictDoNothing()`
- **Global ingestion:** Filings table is global/shared. Events table is firm-scoped.

### Research References (from 02-RESEARCH.md)

- Pattern 2: Submissions API — CIK-based poll, columnar arrays, zip-transpose
- Pattern 3: EFTS Broad Scan — `efts.sec.gov/LATEST/search-index` with `forms` and `dateRange` params
- Pattern 5: Two-Stage Ingestion Flow — Stage 1 writes metadata, Stage 2 fetches content
- Pitfall 2: Submissions API returns columnar arrays (NOT row arrays)
- Pitfall 3: Accession number format mismatch (hyphens for DB, stripped for URLs)
- Pitfall 6: EFTS dateRange requires both startdt and enddt in YYYY-MM-DD

### What to do

Create `apps/api/src/edgar/poll.ts` — the Stage 1 handler for the `edgar_poll` job.

The poll handler runs every 15 minutes via BullMQ cron. It performs two scans:

**Scan A: CIK-based poll for tracked deals**

1. Query `adminDb` for all deals that have a non-null `acquirerCik` or `targetCik`
2. Collect the unique set of CIKs to poll
3. For each CIK, call the submissions API: `https://data.sec.gov/submissions/CIK{paddedCik}.json`
4. Parse the columnar response using zip-transpose (the `recent` field is an object of arrays, NOT an array of objects — see Pitfall 2)
5. Filter by `TRACKED_FORM_TYPES` and by date (only filings since last successful poll, or 30 days for first run)
6. For each new filing: attempt insert with `onConflictDoNothing()` on accession_number

```typescript
import { eq, isNull, isNotNull, or } from 'drizzle-orm';
import { adminDb } from '../db/index.js';
import * as schema from '../db/schema.js';
import { edgarFetch, buildFilingUrl } from './client.js';
import { TRACKED_FORM_TYPES, HIGH_SIGNAL_TYPES, submissionsResponseSchema } from './types.js';
import type { FilingMetadata } from './types.js';

export async function handleEdgarPoll(job: Job): Promise<void> {
  console.log(`[edgar_poll] Starting poll (triggered by: ${job.data.triggeredBy})`);

  // Determine cutoff date: 30 days for first run, 1 hour for subsequent polls
  // (1 hour instead of 15 min provides overlap for safety)
  const lastPollDate = await getLastPollDate();
  const sinceDate = lastPollDate ?? new Date(Date.now() - 30 * 24 * 60 * 60 * 1000);

  // Scan A: CIK-based poll
  const cikFilings = await pollTrackedCiks(sinceDate);

  // Scan B: EFTS broad scan
  const broadFilings = await broadScanForNewDeals(sinceDate);

  // Merge and deduplicate
  const allFilings = deduplicateByAccession([...cikFilings, ...broadFilings]);

  // Insert new filings and enqueue downloads
  let newCount = 0;
  for (const filing of allFilings) {
    const inserted = await insertFilingIfNew(filing);
    if (inserted) {
      newCount++;
      // Enqueue download job for content fetch (Stage 2)
      await enqueueDownload(inserted);
      // Match filing to deals and create events
      await matchAndCreateEvents(inserted, filing);
    }
  }

  console.log(`[edgar_poll] Complete. ${newCount} new filings ingested from ${allFilings.length} total.`);
}
```

**Scan B: EFTS Broad Scan**

```typescript
async function broadScanForNewDeals(since: Date): Promise<FilingMetadata[]> {
  const startdt = since.toISOString().split('T')[0];
  const enddt = new Date().toISOString().split('T')[0];

  // Search for high-signal M&A filing types
  const url = `https://efts.sec.gov/LATEST/search-index?` +
    `q=%22merger+agreement%22&forms=S-4,S-4%2FA,DEFM14A,PREM14A` +
    `&dateRange=custom&startdt=${startdt}&enddt=${enddt}` +
    `&hits.hits._source=period_of_report,entity_name,file_num,form_type,file_date,accession_no`;

  console.log(`[edgar_poll] EFTS broad scan URL: ${url}`);

  try {
    const res = await edgarFetch(url);
    const raw = await res.json();
    const parsed = eftsResponseSchema.safeParse(raw);

    if (!parsed.success) {
      console.warn('[edgar_poll] EFTS response did not match expected schema:', parsed.error.message);
      console.warn('[edgar_poll] Raw EFTS response (first 500 chars):', JSON.stringify(raw).slice(0, 500));
      return [];
    }

    return parsed.data.hits.hits.map(hit => ({
      accessionNumber: hit._source.accession_no,
      filingType: hit._source.form_type,
      filedDate: hit._source.file_date,
      primaryDocument: '', // Will be resolved via index endpoint during download
      filerCik: '', // Not always in EFTS; resolved during download
      filerName: hit._source.entity_name,
    }));
  } catch (err) {
    console.error('[edgar_poll] EFTS broad scan failed:', err);
    return []; // Non-fatal — CIK-based poll is the primary mechanism
  }
}
```

**Key implementation details:**

- `pollTrackedCiks(sinceDate)`: Fetches submissions for each unique CIK. Zips columnar arrays. Filters by form type and date. Returns `FilingMetadata[]`.
- `insertFilingIfNew(filing)`: Uses `adminDb.insert(schema.filings).values({...}).onConflictDoNothing({target: schema.filings.accessionNumber}).returning({id: schema.filings.id})`. Sets `rawContent: null`, `extracted: false`. Sets `status` based on `HIGH_SIGNAL_TYPES`: if the filing type is high-signal, status is `'active'`; otherwise `'pending_review'`.
- `enqueueDownload(filingRow)`: Imports `ingestionQueue` from `../queues/ingestion.js` and calls `ingestionQueue.add('edgar_download', { filingId, accessionNumber, cik, primaryDocument })`.
- `getLastPollDate()`: Query the most recent `filedDate` from the filings table. If no filings exist, return null (triggers 30-day backfill).
- `deduplicateByAccession(filings)`: Deduplicate by accessionNumber before attempting inserts.
- `rawUrl` is constructed from the filing metadata: `buildFilingUrl(cik, accessionNumber, primaryDocument)`. For EFTS results where primaryDocument is empty, use the accession number to construct the filing index URL and resolve later in the download step.
- Import `Job` type from `bullmq`.

Stage 2 handler — fetches HTML content and converts to plain text:

```typescript
import { eq } from 'drizzle-orm';
import type { Job } from 'bullmq';
import { adminDb } from '../db/index.js';
import * as schema from '../db/schema.js';
import { edgarFetch, buildFilingUrl, buildIndexUrl } from './client.js';

export async function handleEdgarDownload(job: Job): Promise<void> {
  const { filingId, accessionNumber, cik, primaryDocument } = job.data;

  console.log(`[edgar_download] Downloading filing ${accessionNumber}`);

  let docUrl: string;

  if (primaryDocument) {
    docUrl = buildFilingUrl(cik, accessionNumber, primaryDocument);
  } else {
    // EFTS results may not have primaryDocument — resolve via index
    const indexUrl = buildIndexUrl(cik, accessionNumber);
    const indexRes = await edgarFetch(indexUrl);
    const indexData = await indexRes.json();
    // The index lists all documents; pick the first HTML/HTM document
    const primaryDoc = resolvePrimaryDocument(indexData);
    if (!primaryDoc) {
      console.warn(`[edgar_download] Could not resolve primary document for ${accessionNumber}`);
      return;
    }
    docUrl = buildFilingUrl(cik, accessionNumber, primaryDoc);
  }

  // Fetch the filing HTML
  const res = await edgarFetch(docUrl);
  const html = await res.text();

  // Convert HTML to plain text
  const { htmlToText } = await import('html-to-text');
  const plainText = htmlToText(html, {
    wordwrap: false,
    selectors: [
      { selector: 'script', format: 'skip' },
      { selector: 'style', format: 'skip' },
      { selector: 'table', options: { uppercaseHeaderCells: false } },
    ],
  });

  // Update filing row with content
  await adminDb
    .update(schema.filings)
    .set({
      rawContent: plainText,
      rawUrl: docUrl, // Update rawUrl in case it was constructed from index resolution
      updatedAt: new Date(),
    })
    .where(eq(schema.filings.id, filingId));

  console.log(`[edgar_download] Stored ${plainText.length} chars for filing ${accessionNumber}`);
}

function resolvePrimaryDocument(indexData: any): string | null {
  // Filing index JSON has a directory.item array with name and type
  try {
    const items = indexData?.directory?.item;
    if (!Array.isArray(items)) return null;
    const htmlDoc = items.find((item: any) =>
      item.name?.endsWith('.htm') || item.name?.endsWith('.html')
    );
    return htmlDoc?.name ?? null;
  } catch {
    return null;
  }
}
```

**File 2: `apps/api/src/edgar/deal-matcher.ts`**

Match filings to deals and auto-create deals for new high-signal filings:

```typescript
import { eq, or, and, isNull } from 'drizzle-orm';
import { adminDb } from '../db/index.js';
import * as schema from '../db/schema.js';
import { HIGH_SIGNAL_TYPES } from './types.js';
import type { FilingMetadata } from './types.js';

interface MatchResult {
  dealId: string | null;
  isNewDeal: boolean;
  firmIds: string[];  // All firms tracking this deal (for event creation)
}

/**
 * Match a filing to existing deals by CIK, or auto-create a new deal for high-signal filings.
 *
 * Returns the matched/created deal ID and all firm IDs that track it.
 * Returns null dealId if filing doesn't match any deal and isn't high-signal.
 */
export async function matchFilingToDeal(filing: FilingMetadata): Promise<MatchResult> {
  // Step 1: Try to match by CIK — check if any deal has this CIK as acquirer or target
  const matchedDeals = await adminDb
    .select({ id: schema.deals.id, firmId: schema.deals.firmId })
    .from(schema.deals)
    .where(
      and(
        isNull(schema.deals.deletedAt),
        or(
          eq(schema.deals.acquirerCik, filing.filerCik),
          eq(schema.deals.targetCik, filing.filerCik),
        ),
      ),
    );

  if (matchedDeals.length > 0) {
    // Filing matches existing deal(s) — return first match and all firm IDs
    const firmIds = [...new Set(matchedDeals.map(d => d.firmId))];
    return {
      dealId: matchedDeals[0].id,
      isNewDeal: false,
      firmIds,
    };
  }

  // Step 2: No CIK match — for high-signal filings, auto-create a deal
  if (HIGH_SIGNAL_TYPES.has(filing.filingType) && filing.filerName) {
    // Auto-created deals are NOT firm-scoped — they need a firm to exist.
    // Since this is a global ingestion stream, auto-created deals are created
    // without a firm_id initially. But deals require firm_id (NOT NULL).
    //
    // Solution: Auto-created deals get a "pending review" status in the filings
    // table (already handled by poll.ts setting status='pending_review' for
    // non-high-signal). For truly new deals from high-signal filings, we
    // cannot auto-create without a firm — log it for review.
    //
    // NOTE: Deal auto-creation requires a firm context. For now, log the
    // discovery and leave dealId as null on the filing. The filing still
    // gets stored and is visible to analysts via the filings endpoint.
    console.log(
      `[deal-matcher] Potential new deal discovered from ${filing.filingType}: ` +
      `${filing.filerName} (CIK: ${filing.filerCik}, accession: ${filing.accessionNumber})`
    );
  }

  return { dealId: null, isNewDeal: false, firmIds: [] };
}
```

Create firm-scoped Event records when filings match deals:

```typescript
import { adminDb } from '../db/index.js';
import * as schema from '../db/schema.js';
import { HIGH_SIGNAL_TYPES } from './types.js';
import type { FilingMetadata } from './types.js';

// Base materiality scores by filing type (from scoring systems in frontend)
const FILING_MATERIALITY: Record<string, number> = {
  'S-4': 80,
  'S-4/A': 75,
  'DEFM14A': 80,
  'PREM14A': 70,
  '8-K': 60,
  '8-K/A': 55,
  'SC 13D': 50,
  'SC 13D/A': 45,
  'SC 13G': 40,
  'SC 13G/A': 35,
  'SC TO-T': 75,
  'SC TO-T/A': 70,
  'SC 14D9': 70,
  'SC 14D9/A': 65,
};

function getSeverity(score: number): 'CRITICAL' | 'WARNING' | 'INFO' {
  if (score >= 70) return 'CRITICAL';
  if (score >= 50) return 'WARNING';
  return 'INFO';
}

/**
 * Create firm-scoped Event records for each firm that tracks this deal.
 *
 * Filings are global, but Events are firm-scoped (have firm_id + RLS).
 * This is the bridge between global ingestion and per-firm visibility.
 */
export async function createFilingEvents(
  filingId: string,
  dealId: string,
  filing: FilingMetadata,
  firmIds: string[],
): Promise<void> {
  if (firmIds.length === 0) return;

  const materialityScore = FILING_MATERIALITY[filing.filingType] ?? 40;
  const severity = getSeverity(materialityScore);

  const eventValues = firmIds.map(firmId => ({
    firmId,
    dealId,
    type: 'FILING' as const,
    subType: filing.filingType,
    title: `${filing.filingType} filed${filing.filerName ? ` by ${filing.filerName}` : ''}`,
    description: `SEC filing ${filing.accessionNumber} (${filing.filingType}) filed on ${filing.filedDate}`,
    source: 'SEC_EDGAR' as const,
    sourceUrl: filing.primaryDocument
      ? `https://www.sec.gov/Archives/edgar/data/${filing.filerCik}/${filing.accessionNumber.replace(/-/g, '')}/${filing.primaryDocument}`
      : `https://www.sec.gov/cgi-bin/browse-edgar?action=getcompany&CIK=${filing.filerCik}&type=${filing.filingType}&dateb=&owner=include&count=10`,
    timestamp: new Date(filing.filedDate),
    materialityScore,
    severity,
    metadata: {
      filingId,
      accessionNumber: filing.accessionNumber,
      filingType: filing.filingType,
      filerCik: filing.filerCik,
      filerName: filing.filerName,
    },
  }));

  if (eventValues.length > 0) {
    await adminDb.insert(schema.events).values(eventValues);
    console.log(`[event-factory] Created ${eventValues.length} FILING events for filing ${filing.accessionNumber}`);
  }
}
```

## must_haves

Derived from phase goal and this plan's scope:

1. **edgar_poll handler processes both CIK-based and EFTS scans** — Without both, coverage is incomplete (EDGAR-01, EDGAR-02)
2. **Columnar submissions response correctly zip-transposed** — Without this, all CIK-based poll data is corrupted (Pitfall 2)
3. **Filing rows inserted with raw_content=null in Stage 1** — Without this, two-stage ingestion is broken (EDGAR-05)
4. **Download handler converts HTML to plain text** — Without this, raw content is unusable for Phase 3 extraction
5. **Filing deduplication by accession number** — Without this, duplicate filings accumulate on every poll cycle
6. **Firm-scoped Event records created from global filings** — Without this, filings don't appear in Inbox for any firm
7. **Worker dispatches to both edgar_poll and edgar_download handlers** — Without this, jobs sit in queue unprocessed
8. **High-signal filings get status 'active'; low-signal get 'pending_review'** — Without this, analyst review queue doesn't work (CONTEXT.md decision)
