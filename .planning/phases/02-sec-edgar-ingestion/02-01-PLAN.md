## Goal

Build the foundational EDGAR infrastructure that all subsequent plans depend on: the rate-limited HTTP client, schema migration to make filings a global table, CIK resolver, and the BullMQ cron job scheduler. After this plan, the system has the ability to make rate-limited calls to SEC EDGAR APIs with the correct User-Agent header, the filings table is a global shared table (no firm_id), deals have CIK columns for polling, and a 15-minute cron schedule is registered.

## Context

### User Decisions (from CONTEXT.md â€” NON-NEGOTIABLE)

- **Global ingestion stream:** Filings table is global/shared (no firm_id, no RLS). Firm-scoped Event records created when a filing matches a firm's deals.
- **Schema change:** Remove firm_id from filings table.
- **Rate limiting:** Must stay under 10 req/s with correct User-Agent header per SEC guidelines.
- **Poll interval:** System-wide 15-minute cron schedule (single job).
- **CIK resolution:** Auto-resolve CIKs from company names via EDGAR company search when a deal is created â€” no manual CIK entry.
- **Filing types tracked:** 8-K, 8-K/A, S-4, S-4/A, DEFM14A, SC 13D, SC 13D/A, SC 13G, SC 13G/A, SC TO, SC TO/A, PREM14A, SC 14D9, SC 14D9/A.

### Research References (from 02-RESEARCH.md)

- Pattern 1: EDGAR HTTP Client with Rate Limiter â€” `limiter` npm package, `RateLimiter` with 9 tokens/second, module-level singleton
- Pattern 4: BullMQ Cron Scheduling â€” `upsertJobScheduler` with `'*/15 * * * *'` cron, idempotent on restart
- Pattern 6: CIK Auto-Resolve â€” `company_tickers.json` from SEC, cache per poll cycle
- Pattern 7: Schema Migration â€” remove firm_id column and RLS policies from filings table
- Anti-patterns: Never call fetch() directly for EDGAR URLs; never fetch company_tickers.json per-company in a loop

**File 2: `apps/api/src/edgar/client.ts`**

Build the rate-limited HTTP client. Key requirements:
- Module-level singleton `RateLimiter` from `limiter` package with 9 tokens/second (not 10 â€” leave headroom per research)
- `User-Agent: j16z admin@j16z.com` on EVERY outbound request
- Exported `edgarFetch(url: string): Promise<Response>` function that ALL EDGAR HTTP calls must go through
- Exported `EDGAR_USER_AGENT` constant for testing

```typescript
import { RateLimiter } from 'limiter';

// Module-level singleton â€” shared across all EDGAR calls in the worker process
const edgarLimiter = new RateLimiter({ tokensPerInterval: 9, interval: 'second' });

export const EDGAR_USER_AGENT = 'j16z admin@j16z.com';

export async function edgarFetch(url: string): Promise<Response> {
  await edgarLimiter.removeTokens(1);
  const res = await fetch(url, {
    headers: {
      'User-Agent': EDGAR_USER_AGENT,
      'Accept': 'application/json',
    },
  });
  if (!res.ok) {
    throw new Error(`EDGAR fetch failed: ${res.status} ${res.statusText} â€” ${url}`);
  }
  return res;
}
```

**File 3: `apps/api/src/edgar/cik-resolver.ts`**

CIK auto-resolution from company names:

```typescript
import { edgarFetch } from './client.js';

interface CompanyTicker {
  cik_str: number;
  ticker: string;
  title: string;
}

// Cache for company_tickers.json â€” refreshed per poll cycle
let tickerCache: Map<string, CompanyTicker> | null = null;
let tickerCacheTimestamp = 0;
const CACHE_TTL_MS = 15 * 60 * 1000; // 15 minutes â€” matches poll interval

async function loadTickerCache(): Promise<Map<string, CompanyTicker>> {
  const now = Date.now();
  if (tickerCache && now - tickerCacheTimestamp < CACHE_TTL_MS) {
    return tickerCache;
  }

  const res = await edgarFetch('https://www.sec.gov/files/company_tickers.json');
  const data = (await res.json()) as Record<string, CompanyTicker>;
  const companies = Object.values(data);

  // Build map keyed by lowercased company name for fast lookup
  const cache = new Map<string, CompanyTicker>();
  for (const company of companies) {
    cache.set(company.title.toLowerCase(), company);
  }

  tickerCache = cache;
  tickerCacheTimestamp = now;
  return cache;
}

export async function resolveCompanyCik(companyName: string): Promise<string | null> {
  const cache = await loadTickerCache();
  const normalized = companyName.toLowerCase().trim();

  // Exact match first
  const exact = cache.get(normalized);
  if (exact) return String(exact.cik_str);

  // Partial match fallback â€” company name contains or is contained by query
  for (const [title, company] of cache) {
    if (title.includes(normalized) || normalized.includes(title)) {
      return String(company.cik_str);
    }
  }

  return null;
}

// Resolve by ticker symbol (for deal board where ticker is known)
export async function resolveCikByTicker(ticker: string): Promise<string | null> {
  const cache = await loadTickerCache();
  const upperTicker = ticker.toUpperCase().trim();

  for (const company of cache.values()) {
    if (company.ticker === upperTicker) {
      return String(company.cik_str);
    }
  }

  return null;
}

// Clear cache (for testing)
export function clearTickerCache(): void {
  tickerCache = null;
  tickerCacheTimestamp = 0;
}
```

**File 4: `apps/api/src/queues/scheduler.ts`**

Register the 15-minute EDGAR poll cron job:

```typescript
import { Queue } from 'bullmq';
import { redisConnection } from './connection.js';

/**
 * Register recurring job schedules.
 *
 * Called ONCE at API server startup (NOT from worker.ts).
 * Uses upsertJobScheduler which is idempotent â€” safe on every restart.
 */
export async function registerSchedules(): Promise<void> {
  const queue = new Queue('ingestion', { connection: redisConnection });

  try {
    // EDGAR poll â€” every 15 minutes (CONTEXT.md locked decision)
    await queue.upsertJobScheduler(
      'edgar-poll-schedule',
      { pattern: '*/15 * * * *' },
      {
        name: 'edgar_poll',
        data: { triggeredBy: 'cron' },
        opts: {
          attempts: 3,
          backoff: { type: 'exponential', delay: 5000 },
        },
      },
    );

    console.log('[j16z-api] Registered edgar_poll schedule (*/15 * * * *)');
  } finally {
    await queue.close();
  }
}
```

## must_haves

Derived from the phase goal: "The system automatically polls EDGAR every 15 minutes, stores raw filings before extraction, and deal board shows real filing metadata."

For this plan specifically (Wave 1 foundation):

1. **EDGAR HTTP client enforces User-Agent on every request** â€” Without this, SEC blocks all requests (EDGAR-03)
2. **Rate limiter enforces <10 req/s** â€” Without this, SEC IP-blocks the system (EDGAR-04)
3. **BullMQ cron registered at 15-minute interval** â€” Without this, no polling happens (EDGAR-01)
4. **Filings table is global (no firm_id)** â€” Without this, the two-stage ingestion design cannot work (CONTEXT.md locked decision)
5. **Deals table has CIK columns** â€” Without this, CIK-based polling has nowhere to look up which companies to track
6. **CIK resolver caches company_tickers.json** â€” Without this, every poll cycle makes O(N) requests to SEC for company lookup
7. **Scheduler NOT imported from worker.ts** â€” Without this, duplicate cron jobs fire on every restart (anti-pattern from research)
