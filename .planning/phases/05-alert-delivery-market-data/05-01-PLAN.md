<objective>
Build the alert delivery infrastructure: schema migration (notification_log table, webhookSecret column, exchangeRatio column), alert worker that evaluates AlertRules against scored events, and three delivery handlers (Resend email, Slack Block Kit, HMAC-signed webhooks).

Purpose: This is the core alert pipeline that turns scored events into push notifications. Without this, analysts must manually check the Inbox for material events.
Output: Working alert_evaluate handler registered in the BullMQ worker, ready to be triggered by event factories in Plan 03.
